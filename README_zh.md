# IDASlicer

IDASlicer 用于把二进制中的某些部分（函数、段或任意选区）提取、"切片"到新的 IDA 数据库或 `.seg` 文件中，方便从大型二进制里切出可单独分析的一块。

[English Version](./README.md)

## 功能特点

- **加入切片列表**（在反汇编视图右键 → `Add to Slicer/`）：
  - **Function**、**Selection** 或 **Current segment**。
  - **Function recursively**（递归添加）：加入该函数，以及它引用的所有代码/数据范围，并递归跟进。被引用的库函数和导入桩（thunk）只取其第一条指令，这样调用它们时能解析出名字，而不是显示为未知地址。
- **切片管理面板**（`Edit → Plugins → IDASlicer`）：检查、编辑（名称、范围、权限、类型、对齐）或删除条目；Size 列显示每个范围的大小。编辑一个递归条目的范围时会对新范围重新扫描引用。列表按输入文件 MD5 保存在 `idaslicer_config.json` 中。
- **创建 IDA 数据库 (9.1+)**：生成只包含所选切片的新 `.i64`，并按文件类型（ELF/PE/Mach-O）和架构（x86/x64/ARM/ARM64）自动选择模板。
- **导出 `.seg` 文件**：每个条目连同段名、范围、权限、类型/对齐、字节以及用户定义的符号名一起保存。可选择**合并**为单个文件，便于批量传输与导入。
- **导入 `.seg` 文件 (9.1+)**：把单个或合并的文件导回数据库，恢复字节、段属性和符号名。与现有段重叠时，可覆盖数据或在空隙处创建新段。

## 要求

- **IDA Pro 9.1+**：创建/导入数据库功能需要（使用 `ida_domain` API）。其他功能在更早的 9.x 上也可用。
- **PySide6**：现代 IDA Pro 已自带。

## 安装

将整个文件夹复制到 IDA 插件目录：
- **Windows**: `%AppData%\Hex-Rays\IDA Pro\plugins`
- **Linux/macOS**: `~/.idapro/plugins`

## 工作原理

- **数据库切片**：从 `obj_minis/` 复制一个迷你模板，再通过 `ida_domain` API 运行后台进程，在不关闭当前会话的情况下重建段、字节、属性与名称。
- **`.seg` 文件**：是 pickle 序列化的字典，保存元数据、字节和收集到的符号名，因此不依赖脆弱的文件名解析。导入时自动区分单个文件与合并文件，并把所有信息重新应用（用 `ida_name` 设置名称、在新建段上设置类型/对齐），同时处理重叠冲突。
